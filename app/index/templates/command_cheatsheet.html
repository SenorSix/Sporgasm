My Frequently Used Commands

=== Local File Operations (Laptop) ===
scp -r Sporgasm_v3 user@ip:/home/user/             - Copy folder to server (run from local, be in correct dir)
cd /path/to/dir                                    - Change into directory
ls /path/to/dir                                    - List contents of directory
tree -L 2                                          - Tree view to level 2 (overwhelming past that)
cat .gitignore                                     - View contents of .gitignore
touch filename.py                                  - Create blank file
script session.log (type exit or Ctrl+D to exit)   - Record terminal session

=== Server File Operations ===
sudo tree /var/www                                 - Show contents of /var/www
rm phpBB-3.3.10.zip                                - Removes a file
rm phpBB-3.3.10.zip*                               - remove all files with phpBB-3.3.10.zip (* matches anything after filename)
sudo rm -rf /var/www/sporgasm                      - Delete a folder (sporgasm example)
sudo cp -r /var/www/html/project /var/www/backup   - Copy folder to backup location
sudo mv /path/to/file /destination/path            - Move file/folder on server

Git & Version Control

git init                             # Initialize new Git repo
git add .                            # Stage all changes
git commit -m "message"              # Commit changes
git branch -M main                   # Rename current branch to 'main' (do early)
git remote add origin <url>    # Link to remote repo (GitHub, etc.)
git remote -v                        # View remote URLs
git push -u origin main              # Push to remote repo
git status                           # Check repo status
git filter-repo --force --path credentials.json --invert-paths
                                                   - Strip file from commit history (NOTE: ask AI before reusing)
touch .gitignore                                   - Create gitignore file
nano .gitignore                                    - Open gitignore file in order to edit. venv/ ignores venv for example
git reset --soft HEAD~1                            - Uncommits the last commit but leave all changes staged (so you don't lose your work) so you can change commit message or whatever

Deployment to Server

mv /var/www/sporgasm /var/www/sporgasm_backup      - Create a backup
git clone <url> sporgasm (folder pushing to)       - Push from git to server if creating a backup
git pull origin main                               - Push from git to server if just updating something
to own sporgasm, cd /var/www, then: sudo chown -R lana_burismo:lana_burismo sporgasm (if sudo cloned)
                                                   - Gives lana ownership of sporgasm
cd sporgasm or cd /var/www/sporgasm                - Go to sporgasm folder
python3 -m venv venv                               - Install venv
source venv/bin/activate                           - Activate venv
pip install -r requirements.txt                    - Install requirements
gunicorn -b 127.0.0.1:8000 run:app                 - Runs the application on 127.0.0.1:8000 (localhost) and will allw you to test the app without deploying to a live server
sudo systemctl restart sporgasm                    - Restart Sporgasm
sudo systemctl status sporgasm                     - Ensure Gunicorn is running correctly after restart

=== Deployment & Apache ===
sudo systemctl restart apache2                     - Restart Apache
sudo chown -R www-data:www-data /var/www/html      - Fix Apache ownership for web files

=== System Maintenance (Linux) ===
sudo apt update                                    - Update package lists
sudo apt dist-upgrade                              - Full upgrade, handles dependencies

=== Security ===
sudo apt install unattended-upgrades               - Install auto-security updates
sudo dpkg-reconfigure --priority=low unattended-upgrades
                                                   - Configure how unattended-upgrades behaves

=== Sed & One-Liners ===
sed -i 's|/red/|/|g' /var/www/html/forum/cache/themes/theme17/*.css
                                                   - Replace all `/red/` with `/` in CSS files (NOTE: ask AI before using again)

Additional Suggested Commands

=== File Permissions & Ownership ===
chmod +x filename                                   - Make file executable
chown user:group file                               - Change ownership of file
ls -la                                              - Show permissions and hidden files

=== Python & Virtual Environments ===
python3 -m venv venv                                - Create a virtual environment
source venv/bin/activate                            - Activate virtual environment
deactivate                                          - Exit virtual environment
pip install -r requirements.txt                     - Install packages from requirements

=== Flask & Blueprint Tips ===
flask run                                           - Run a Flask app (if configured)

clear                                               - Clear terminal screen

=== Git Best Practices ===
echo filename >> .gitignore                         - Add file to .gitignore
git log --oneline                                   - View concise commit history
git diff                                            - View file changes before commit

Let's Encrypt

=== Installing and Setting Up Cron Job ===
sudo apt update
sudo apt install certbot python3-certbot-apache
sudo crontab -l                                     - Check for existing cron job
should see "0 */12 * * * certbot renew --quiet"

=== If Not Setup ===
sudo crontab -e     then select nano (probably 1)
0 */12 * * * certbot renew --quiet      add this line into the file
                                                    - Renews the certificates every 12 hours
systemctl list-timers | grep certbot                - Checks if the systemd timer is active (should show "certbot.timer")
sudo systemctl enable certbot.timer                 - Enables certbot timer
sudo systemctl start certbot.timer                  - Starts certbot timer
sudo cerbot renew --dry-run                         - Tests renewal process

HTML

=== Basic HTML Structure ===
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Title</title>
</head>
<body>
    <!-- Content Goes Here -->
</body>
</html>

=== Creating a Link ===
<a href="https://example.com" target="_blank" rel="noopener noreferrer">Click Me!</a>

=== Image Tag ===
<img src="image.jpg" alt="Description of the image">

=== Table Example ===
<table>
    <tr>
        <th>Header 1</th>
        <th>Header 2</th>
    </tr>
    <tr>
        <td>Row 1, Col 1</td>
        <td>Row 1, Col 2</td>
    </tr>
</table>

=== Form Example ===
<form action="/submit" method="POST">
    <input type="text" name="username" placeholder="Enter username" required>
    <input type="password" name="password" placeholder="Enter password" required>
    <button type="submit">Submit</button>
</form>

CSS Commands & Syntax


Basic Style
body {
    font-family: Arial, sans-serif;
    margin: 0;
    padding: 0;
    background-color: #f4f4f4;
}

Text Styling
h1 {
    color: #61dafb;
    font-size: 2em;
    text-align: center;
}

Positioning an Element
.container {
    width: 100%;
    position: relative;
}
.child {
    position: absolute;
    top: 10px;
    left: 20px;
}

Flexbox Example
.flex-container {
    display: flex;
    justify-content: center;
    align-items: center;
}

Grid Layout Example
.grid-container {
    display: grid;
    grid-template-columns: repeat(3, 1fr);
    gap: 10px;
}
.grid-item {
    background-color: lightgray;
    padding: 20px;
}

Hover Effect
a:hover {
    color: red;
}

Reset some defaults
* {
    margin: 0;
    padding: 0;
    box-sizing: border-box;
}

Centering Content
.center {
    display: flex;
    justify-content: center;
    align-items: center;
    height: 100vh;
}

Docker Commands


docker --version                                    - Show Docker version
docker ps                                           - List running containers
docker ps -a                                        - List all containers (including stopped ones)
docker images                                       - List Docker images
docker build -t <image-name> .                      - Build a Docker image
docker run -d <image-name>                          - Run a container in detached mode
docker exec -it <container-id> bash                 - Access running container
docker-compose up                                   - Start containers defined in a docker-compose.yml
